# 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
kt-blog（技術ブログシステム）

### 1.2 目的
個人が学習した技術内容を記事として投稿・管理し、広く公開することで知識の共有を行うためのブログシステムを構築する。

### 1.3 対象ユーザー
- **閲覧者**: インターネット上の全ユーザー（認証不要）
- **管理者**: ブログ運営者（認証必要）

## 2. 機能要件

### 2.1 公開機能（閲覧者向け）

#### 2.1.1 記事一覧表示
- **FR-001**: 公開済み記事の一覧を表示できる
- **FR-002**: 記事は新しい順に表示される
- **FR-003**: ページネーション機能を提供する（1ページ10件）
- **FR-004**: 各記事にタイトル、サムネイル、概要、投稿日、カテゴリ、タグを表示する

#### 2.1.2 記事詳細表示
- **FR-005**: 記事の全文をマークダウン形式でレンダリングして表示する
- **FR-006**: 記事に紐づくカテゴリとタグを表示する
- **FR-007**: 記事に投稿されたコメントを表示する
- **FR-008**: シンタックスハイライト機能を提供する（コードブロック）

#### 2.1.3 カテゴリ機能
- **FR-009**: カテゴリ一覧を表示できる
- **FR-010**: カテゴリを選択すると、そのカテゴリの記事一覧を表示する
- **FR-011**: 各カテゴリの記事数を表示する

#### 2.1.4 タグ機能
- **FR-012**: タグクラウドまたはタグ一覧を表示できる
- **FR-013**: タグを選択すると、そのタグが付いた記事一覧を表示する
- **FR-014**: 各タグの記事数を表示する

#### 2.1.5 検索機能
- **FR-015**: キーワードで記事を検索できる
- **FR-016**: タイトル、本文、カテゴリ、タグが検索対象となる

#### 2.1.6 コメント機能
- **FR-017**: 記事に対してコメントを投稿できる
- **FR-018**: コメント投稿時に名前とメールアドレスを入力する（メール非公開）
- **FR-019**: コメントは管理者承認後に公開される

### 2.2 管理機能（管理者向け）

#### 2.2.1 認証機能
- **FR-020**: メールアドレスとパスワードでログインできる
- **FR-021**: JWTトークンによるセッション管理を行う
- **FR-022**: ログアウト機能を提供する
- **FR-023**: パスワードは暗号化して保存する

#### 2.2.2 記事管理
- **FR-024**: 新規記事を作成できる
- **FR-025**: 既存記事を編集できる
- **FR-026**: 記事を削除できる
- **FR-027**: 記事の公開/非公開を切り替えられる
- **FR-028**: マークダウンエディタで記事を執筆できる
- **FR-029**: 記事プレビュー機能を提供する
- **FR-030**: 記事にカテゴリを1つ設定できる
- **FR-031**: 記事に複数のタグを設定できる
- **FR-032**: 記事にサムネイル画像を設定できる

#### 2.2.3 カテゴリ管理
- **FR-033**: カテゴリを作成できる
- **FR-034**: カテゴリを編集できる（名前、説明）
- **FR-035**: カテゴリを削除できる（紐づく記事がない場合）

#### 2.2.4 タグ管理
- **FR-036**: タグを作成できる
- **FR-037**: タグを編集できる（名前）
- **FR-038**: タグを削除できる（紐づく記事がない場合）

#### 2.2.5 コメント管理
- **FR-039**: 投稿されたコメント一覧を表示できる
- **FR-040**: コメントを承認/非承認できる
- **FR-041**: コメントを削除できる

#### 2.2.6 画像管理
- **FR-042**: 画像ファイルをアップロードできる（jpg, png, gif）
- **FR-043**: アップロードした画像一覧を表示できる
- **FR-044**: 画像のURLをコピーして記事に挿入できる
- **FR-045**: 画像を削除できる

## 3. 非機能要件

### 3.1 パフォーマンス
- **NFR-001**: 記事一覧の表示は3秒以内に完了すること
- **NFR-002**: 記事詳細の表示は2秒以内に完了すること

### 3.2 セキュリティ
- **NFR-003**: パスワードはbcryptで暗号化すること
- **NFR-004**: 管理画面へのアクセスは認証必須とすること
- **NFR-005**: XSS、CSRF攻撃への対策を実装すること
- **NFR-006**: SQLインジェクション対策を実装すること

### 3.3 可用性
- **NFR-007**: システムは24時間365日稼働することを目指す
- **NFR-008**: データベースのバックアップを定期的に取得する

### 3.4 保守性
- **NFR-009**: コードはTypeScriptで型安全に記述すること
- **NFR-010**: ESLint/Prettierでコード品質を保つこと
- **NFR-011**: 適切なコメントとドキュメントを作成すること

### 3.5 ユーザビリティ
- **NFR-012**: レスポンシブデザインでモバイル対応すること
- **NFR-013**: 直感的でわかりやすいUIを提供すること

## 4. 制約条件

### 4.1 技術的制約
- フロントエンド: Vue.js 3 + Nuxt.js 3
- バックエンド: Nest.js
- データベース: MySQL 8.0
- TypeScriptを使用すること

### 4.2 開発期間
- Phase 1（MVP）: 基本的な記事CRUD機能
- Phase 2: カテゴリ・タグ機能
- Phase 3: コメント・画像アップロード機能
- Phase 4: 検索機能・UI改善

## 5. システム境界

### 5.1 対象範囲
- 記事管理システム全般
- 管理画面
- 公開フロント画面

### 5.2 対象外
- ユーザー登録機能（管理者のみ固定）
- SNS連携
- アクセス解析
- メール配信機能

## 6. 用語定義

| 用語 | 説明 |
|------|------|
| 記事 | ブログに投稿される技術記事 |
| カテゴリ | 記事を分類するための大分類 |
| タグ | 記事に付与するキーワード |
| 公開/非公開 | 記事の表示状態 |
| マークダウン | 記事を記述するための軽量マークアップ言語 |
| 管理者 | ブログの管理権限を持つユーザー |
